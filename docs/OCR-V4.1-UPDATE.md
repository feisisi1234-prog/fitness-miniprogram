# OCR识别功能 v4.1 更新说明

## 📋 更新概述

针对用户反馈的实际使用场景，进行了针对性优化，特别是对复杂格式和连续文本的支持。

## 🎯 核心改进

### 1. 智能文本分割

**问题**：
- OCR识别出的文本可能是一大段连续的文本，没有换行符
- 之前的逻辑依赖换行符来分隔不同的动作
- 导致识别失败或只识别到第一个动作

**解决方案**：
```javascript
// 如果文本很长且没有换行符，尝试智能分割
if (lines.length === 1 && processedText.length > 100) {
  processedText = processedText
    .replace(/(\d+\s*秒)/g, '$1\n')  // 在"秒"后换行
    .replace(/([一-龥]{2,8})\s+(\d+\s*[组組])/g, '\n$1 $2')  // 在动作名称前换行
    .replace(/休息\s*\d+\s*秒/g, match => match + '\n');  // 在休息时间后换行
}
```

**效果**：
- 自动将连续文本分割成多行
- 每个动作独立识别
- 提高识别准确度

### 2. 支持简短标题

**问题**：
- 用户使用简短的部位名称作为标题（如"胸"、"背"、"腿"）
- 之前的逻辑要求标题至少2个字
- 导致标题识别失败

**解决方案**：
```javascript
const titlePatterns = [
  /^([胸背腿肩臂腹][部]?)\s/,  // 单字部位：胸、背、腿、肩、臂、腹
  /^([一-龥]{1,20})(?:\s|$)/  // 行首的中文（1-20个字）
];
```

**支持的标题**：
- 单字：胸、背、腿、肩、臂、腹
- 两字：胸部、背部、腿部、肩部、臂部、腹部
- 多字：全身力量训练、HIIT燃脂训练等

### 3. 支持组内暂停格式

**问题**：
- 用户使用"10+5"表示组内暂停（10次+暂停+5次）
- 之前的逻辑无法识别这种格式
- 导致次数识别错误

**解决方案**：
```javascript
const repsPatterns = [
  /[xX×*]\s*(\d+)(?:\+\d+)?/, // x后面的数字，支持10+5格式
  /(\d+)\+\d+/, // 组内暂停格式：10+5
  // ...
];
```

**支持的格式**：
```
✅ 3组 × 12次
✅ 3组 × 10+5次
✅ 3 × 10+5
✅ 3x10+5
```

**识别逻辑**：
- 提取"+"前面的数字作为次数
- 例如"10+5"识别为10次
- 保留完整信息供用户参考

### 4. 扩展胸部和背部动作库

**新增胸部动作**：
- 坐姿下斜推
- 上斜推
- 平板推
- 飞鸟
- 绳索夹胸

**新增背部动作**：
- 高位下拉
- 坐姿划船
- 杠铃划船
- 直臂下压

**总动作数**：
- v4.0: 30+个动作
- v4.1: 40+个动作

### 5. 优化长文本处理

**问题**：
- 之前跳过长度超过50的行
- 但OCR可能将多个动作识别为一行
- 导致动作丢失

**解决方案**：
- 移除长度限制
- 使用智能分割处理长文本
- 只跳过明确的标题行

### 案例2：连续文本

**输入**（无换行符）：
```
处理者1行：分化 1 胸 坐姿下斜推 3 组 × 12 次 组内暂停 10+5 休息 60 秒吐铃飞鸟 ocr_parser.js? [sm]:139 处理者1行：分化 1 胸 坐姿下斜推 3 组 × 12 次 组内暂停 10+5 休息 60 秒吐铃飞鸟 3 组 × 15 次 无间歇 上斜绳索夹胸 3 组 × 15 次 无休息 背 主训 钢铃下拉两段变式 3 组 × 12 次 后仰位和直立位 休息 60 秒
```

**处理过程**：
1. 检测到长文本（>100字符）
2. 在"秒"后自动换行
3. 在动作名称前自动换行
4. 分割成多行独立处理

**预期输出**：
```

类别: 力量训练
动作:
1. 坐姿下斜推 - 3组 x 10次, 休息60秒

```

## 🔧 使用建议

### 对于用户

1. **使用简短标题**
   - 可以使用单字标题：胸、背、腿、肩、臂、腹
   - 系统会自动识别

2. **组内暂停格式**
   - 使用"10+5"表示组内暂停
   - 系统会提取第一个数字作为次数
   - 完整信息会保留在原始文字中

3. **确保动作分行**
   - 虽然系统支持连续文本
   - 但分行格式识别更准确
   - 推荐每行一个动作

4. **使用标准动作名称**
   - 查看支持的动作列表
   - 使用标准名称或常见别名
   - 提高识别准确度

### 对于开发者

1. **测试智能分割**
   - 使用 `utils/test-ocr-parser.js` 测试
   - 验证长文本分割效果
   - 调整分割规则

2. **添加新动作**
   - 编辑 `utils/ocr-parser.js`
   - 在 `exerciseDatabase` 中添加
   - 包含多个变体和别名

3. **优化分割规则**
   - 根据实际OCR输出调整
   - 添加更多分割模式
   - 提高分割准确度
