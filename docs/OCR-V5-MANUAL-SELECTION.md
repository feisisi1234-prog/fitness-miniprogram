# OCR V5 更新：手动文字选择功能

## 更新日期
2026年2月24日

## 更新内容

### 新增功能：手动文字选择

当 OCR 自动识别质量较低或格式不符合要求时，用户现在可以手动选择识别出的文字行来创建训练计划。

## 功能特点

### 1. 智能触发
系统自动检测 OCR 质量，当满足以下任一条件时提示用户手动选择：
- OCR 质量评分 < 50 分
- 未识别到任何动作
- 识别结果为默认占位动作

### 2. 友好的选择界面
- ✅ 清晰的文字行列表
- ✅ 复选框选择机制
- ✅ "全选"和"清除"快捷按钮
- ✅ 选中行高亮显示（绿色）
- ✅ 实时视觉反馈

### 3. 灵活的操作
- 点击任意行切换选中状态
- 一键全选所有文字
- 一键清除所有选择
- 随时取消操作

### 4. 智能解析
选中的文字会被重新组合并解析，支持：
- 标准格式：`深蹲 3组 x 12次 休息60秒`
- 简化格式：`深蹲 3 12 60`
- 组内暂停：`坐姿下斜推 3组 x 12次 10+5 休息60秒`
- 简单标题：`胸`、`腿`、`背`

## 使用流程

```
1. 用户点击"📷 扫描图片"
   ↓
2. 选择/拍摄图片
   ↓
3. OCR 自动识别文字
   ↓
4. 系统评估识别质量
   ↓
5a. 质量良好 → 直接显示解析结果
5b. 质量较低 → 提示"手动选择"或"查看结果"
   ↓
6. 用户选择"手动选择"
   ↓
7. 显示文字选择界面
   ↓
8. 用户选择需要的文字行
   ↓
9. 点击"确认创建"
   ↓
10. 重新解析并显示结果
```

## 代码更新

### custom-plans.js
新增函数：
- `showTextSelection(originalText)` - 显示文字选择界面
- `toggleTextLineSelection(e)` - 切换行选中状态
- `selectAllTextLines()` - 全选
- `clearAllTextLines()` - 清除选择
- `confirmTextSelection()` - 确认选择
- `cancelTextSelection()` - 取消选择

修改函数：
- `showRecognizedPlan()` - 添加质量检测和手动选择提示

新增数据字段：
```javascript
{
  showTextSelectionModal: false,  // 显示选择弹窗
  ocrOriginalText: '',            // 原始文字
  ocrTextLines: []                // 文字行数组
}
```

### custom-plans.wxml
新增组件：
- 文字选择弹窗 (`.text-selection-modal`)
- 文字行列表 (`.text-line-item`)
- 选择操作按钮 (`.selection-actions`)

### custom-plans.wxss
新增样式：
- `.text-selection-modal` - 弹窗容器
- `.text-line-item` - 文字行样式
- `.text-line-item.selected` - 选中状态样式
- `.selection-action-btn` - 操作按钮样式

## 用户体验改进

### 之前
- OCR 识别失败 → 用户只能重新拍照或手动输入
- 格式不规范 → 识别结果不准确，用户需要手动修改
- 混合内容 → 无法过滤无关文字

### 现在
- OCR 识别失败 → 用户可以手动选择有效文字行
- 格式不规范 → 用户可以选择正确的文字行重新解析
- 混合内容 → 用户可以只选择训练相关的文字

## 技术亮点

1. **质量评估系统**：`evaluateOCRQuality()` 自动评估识别质量
2. **智能分行**：自动按换行符分割文字，保持原始结构
3. **状态管理**：每行文字独立的选中状态
4. **视觉反馈**：选中行实时高亮，用户操作清晰可见
5. **容错机制**：即使选择失败，仍可返回查看原始结果

## 已知限制

1. 文字行分割依赖换行符，如果 OCR 结果没有换行符，可能需要手动处理
2. 选择界面不支持编辑文字内容，只能选择/取消选择
3. 不支持拖拽排序或合并文字行

## 未来改进方向

1. **智能预选**：根据关键词自动预选可能的训练动作行
2. **行内编辑**：允许用户直接修改文字行内容
3. **拖拽排序**：支持调整文字行顺序
4. **合并/分割**：支持合并或分割文字行
5. **历史记录**：保存用户的选择历史，方便下次使用
