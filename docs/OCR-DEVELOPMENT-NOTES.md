# OCR功能开发说明

## 🔧 技术实现

### 1. OCR API说明

本功能使用微信小程序的OCR文字识别API：

```javascript
wx.ocr.recognize({
  path: imagePath,        // 图片路径
  ocrType: 'general',     // 识别类型：general(通用)
  success: (res) => {},   // 成功回调
  fail: (err) => {}       // 失败回调
})
```

### 2. API可用性

**真机环境**：
- ✅ 支持OCR识别
- ✅ 需要网络连接
- ✅ 识别速度较快（1-3秒）

**开发工具**：
- ❌ 不支持OCR API
- ✅ 自动使用模拟识别
- ✅ 提供多个模板选择

### 3. 错误处理

代码中实现了完善的错误处理：

```javascript
// 检查API是否可用
if (!wx.ocr || typeof wx.ocr.recognize !== 'function') {
  // 使用模拟识别
  this.simulateOCR(imagePath);
  return;
}

// API调用失败时的处理
wx.ocr.recognize({
  // ...
  fail: (err) => {
    // 自动降级到模拟识别
    this.simulateOCR(imagePath);
  }
});
```

## 📝 文字解析逻辑

### 解析流程

```
原始文字
  ↓
清理空白字符
  ↓
提取标题
  ↓
识别类别
  ↓
提取动作列表
  ↓
解析组数/次数/休息时间
  ↓
生成计划对象
```

### 支持的格式

**标准格式**：
```
动作名称 组数组 x 次数次 休息时间秒
例：深蹲 3组 x 12次 休息60秒
```

**简化格式**：
```
动作名称 组数x次数 时间s
例：深蹲 3x12 60s
```

**中文格式**：
```
动作名称 组数组次数次
例：深蹲 3组12次
```

### 正则表达式

```javascript
// 提取组数
/(\d+)\s*[组次]/

// 提取次数
/[xX×]\s*(\d+)|(\d+)\s*[次个]/

// 提取休息时间
/(?:休息|间歇)?\s*(\d+)\s*[秒s]/i
```

## 🐛 常见问题

### Q1: 开发工具报错 "Cannot read property 'recognize' of undefined"

**原因**：开发工具不支持OCR API

**解决**：代码已自动处理，会使用模拟识别

### Q2: 真机识别失败

**可能原因**：
- 网络问题
- 图片不清晰
- 文字格式不标准

**解决方法**：
- 检查网络连接
- 重新拍照
- 使用"修改"功能手动调整

### Q3: 识别结果不准确

**优化方法**：
1. 使用标准格式
2. 确保文字清晰
3. 避免复杂背景
4. 使用"修改"功能调整

### Q4: 如何添加新的动作

**方法1**：修改 `utils/ocr-parser.js`
```javascript
const exerciseNames = [
  // 添加新动作
  '新动作名称',
  // ...
];
```

**方法2**：识别后手动添加
- 点击"修改"
- 点击"添加动作"
- 输入新动作信息
