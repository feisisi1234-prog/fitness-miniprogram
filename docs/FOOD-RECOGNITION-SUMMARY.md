## 🎨 UI 设计

### 页面状态

1. **初始状态** - 上传区域
   - 大图标 📷
   - 提示文字："轻触此处拍照"
   - 副标题："也可以从相册选择图片"

2. **预览状态** - 已选择图片
   - 图片预览（600rpx 高度）
   - 两个按钮："重新选择" 和 "开始识别"

3. **识别中状态** - 加载遮罩
   - 旋转的加载动画
   - 提示文字："AI 正在识别中..."
   - 副标题："请稍候，这可能需要几秒钟"

4. **结果状态** - 识别完成
   - 缩小的图片预览（400rpx 高度）
   - 结果卡片：
     - 食物名称（大字体，绿色）
     - 3个网格：热量、GI、GL
     - 食用建议（黄色背景）
   - 底部按钮："识别其他食物"

### 颜色方案
- 主色：绿色 (#07c160) - 健康、营养
- 辅助色：蓝色 (#007aff) - 科技、AI
- 背景：渐变 (#f5f7fa → #c3cfe2)
- 卡片：白色渐变 (#ffffff → #f8f9fa)

## 🔧 技术实现

### API 集成流程

```javascript
// 1. 上传文件
uploadFile(filePath) → fileId

// 2. 创建 Chat
createChat(fileId) → { chatId, conversationId }

// 3. 轮询消息
pollMessages(conversationId, chatId) → result

// 4. 解析结果
parseRecognitionResult(content) → {
  name, calories, gi, gl, suggestion
}
```

### 性能监控点

```
流程开始 → 选择图片 → 上传文件 → 创建Chat → 轮询消息 → 显示结果
   ↓          ↓          ↓          ↓          ↓          ↓
  0ms      用户操作    1-2秒     0.2-0.5秒   2-5秒     即时
```

### 错误处理策略

1. **网络错误** → 显示友好提示，允许重试
2. **超时错误** → 30秒后自动超时，提示用户
3. **解析错误** → 显示原始内容，方便调试
4. **API 错误** → 显示具体错误信息

## 📱 用户流程

```
1. 用户打开小程序
   ↓
2. 点击首页"食物识别"卡片
   ↓
3. 进入食物识别页面
   ↓
4. 点击上传区域
   ↓
5. 选择"拍照"或"从相册选择"
   ↓
6. 选择/拍摄食物图片
   ↓
7. 查看图片预览
   ↓
8. 点击"开始识别"
   ↓
9. 等待识别（3-8秒）
   ↓
10. 查看识别结果
    - 食物名称
    - 热量
    - 升糖指数
    - 升糖负荷
    - 食用建议
   ↓
11. 点击"识别其他食物"继续使用
```
