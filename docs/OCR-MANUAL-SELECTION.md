# OCR 手动文字选择功能

## 功能概述

当 OCR 自动识别质量较低时，用户可以手动选择识别出的文字行来创建训练计划。

## 触发条件

手动文字选择功能在以下情况下会自动触发：

1. **OCR 质量评分 < 50 分**
2. **未识别到任何动作** (`exercises.length === 0`)
3. **识别结果为默认动作** (动作名为"动作1")

## 用户流程

### 1. 扫描图片
用户点击"📷 扫描图片"按钮，选择或拍摄包含训练计划的图片。

### 2. OCR 识别
系统自动调用 OCR API 识别图片中的文字。

### 3. 质量评估
系统评估识别质量：
- **质量良好 (≥50分)**: 直接显示解析结果
- **质量较低 (<50分)**: 提示用户选择手动选择或查看结果

### 4. 手动选择界面
如果用户选择"手动选择"，会看到：
- 所有识别出的文字行
- 每行前面有复选框
- "全选"和"清除"快捷按钮
- 选中的行会高亮显示（绿色背景）

### 5. 选择文字
用户可以：
- 点击任意行来选中/取消选中
- 点击"全选"选中所有行
- 点击"清除"取消所有选择

### 6. 确认创建
点击"确认创建"后：
- 系统将选中的文字行组合成新文本
- 重新调用解析器解析训练计划
- 显示解析结果供用户确认

## 技术实现

### 数据结构

```javascript
{
  showTextSelectionModal: false,  // 是否显示文字选择弹窗
  ocrOriginalText: '',            // 原始 OCR 文字
  ocrTextLines: [                 // 文字行数组
    {
      id: 0,
      text: '深蹲 3组 x 12次 休息60秒',
      selected: false
    },
    // ...
  ]
}
```

### 核心函数

#### `showTextSelection(originalText)`
显示文字选择界面，将原始文字按行分割并初始化选择状态。

#### `toggleTextLineSelection(e)`
切换单行文字的选中状态。

#### `selectAllTextLines()`
选中所有文字行。

#### `clearAllTextLines()`
清除所有选择。

#### `confirmTextSelection()`
确认选择，将选中的文字行组合并重新解析。

#### `cancelTextSelection()`
取消选择，关闭弹窗。

## UI 设计

### 弹窗样式
- 半透明遮罩层
- 白色圆角卡片
- 占屏幕 90% 宽度，最高 70% 高度
- 渐变背景 (#fff → #f8f9fa)

### 文字行样式
- 默认：灰色背景 (#f8f9fa)
- 选中：绿色背景 (rgba(7, 193, 96, 0.1))
- 选中边框：绿色 (#07c160)
- 选中文字：绿色加粗

### 按钮样式
- "全选"/"清除"：渐变绿蓝色按钮
- "取消"：灰色按钮
- "确认创建"：渐变绿蓝色按钮

## 使用场景

### 场景 1：手写文字
用户拍摄手写的训练计划，OCR 识别质量较低，系统自动提示手动选择。

### 场景 2：格式不规范
用户的训练计划格式不符合标准格式，自动解析失败，用户可以手动选择有效的文字行。

### 场景 3：混合内容
图片中包含训练计划和其他无关文字，用户可以只选择训练相关的文字行。

### 场景 4：部分识别错误
OCR 识别了大部分内容，但有几行错误，用户可以取消选择错误的行。

## 优势

1. **提高成功率**：即使 OCR 质量不佳，用户仍可创建计划
2. **用户控制**：用户可以精确控制哪些文字被用于创建计划
3. **灵活性**：支持各种格式和内容的图片
4. **易用性**：简单的点击操作，无需手动输入

## 改进建议

### 未来可能的增强功能：
1. **智能预选**：根据关键词自动预选可能的训练动作行
2. **行编辑**：允许用户直接编辑文字行内容
3. **拖拽排序**：允许用户调整文字行顺序
4. **合并行**：允许用户合并多行文字
5. **分割行**：允许用户将一行文字分割成多行
