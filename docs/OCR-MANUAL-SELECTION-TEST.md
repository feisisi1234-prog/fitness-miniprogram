# OCR 手动文字选择功能测试指南

## 测试目的
验证手动文字选择功能是否正常工作，确保用户可以在 OCR 识别质量较低时手动选择文字创建训练计划。

## 测试环境
- 微信开发者工具
- 真机测试（可选）

## 测试步骤

### 测试 1：使用模拟 OCR 测试手动选择

1. **打开自定义计划页面**
   - 点击底部导航栏的"自定义计划"

2. **点击"📷 扫描图片"按钮**
   - 会弹出拍照提示，点击"继续"

3. **选择图片**
   - 在开发者工具中，会自动进入模拟 OCR 模式
   - 选择"测试手动选择（低质量文字）"选项

4. **验证触发条件**
   - 应该看到弹窗：标题"识别质量较低"
   - 内容："自动识别效果不佳，建议手动选择文字创建计划。是否查看识别的文字并手动选择？"
   - 两个按钮："手动选择"和"查看结果"

5. **点击"手动选择"按钮**
   - 应该看到文字选择弹窗
   - 弹窗标题："选择文字创建计划"
   - 提示信息："💡 点击选择需要的文字行，然后确认创建计划"

6. **验证文字行显示**
   - 应该看到多行文字，每行前面有复选框
   - 文字内容应该包括：
     - "胸部训练"
     - "坐姿下斜推 3组 12次 组内暂停 10+5 休息 60"
     - "上斜推 3组 12次 休息 60"
     - "平板推 3组 12次 休息 60"
     - "飞鸟 3组 15次 休息 45"
     - "绳索夹胸 3组 15次 休息 45"
     - "这是一些无关的文字"
     - "可能还有一些干扰内容"
     - "俯卧撑 3组 力竭 休息 60"

7. **测试选择功能**
   - 点击任意一行，该行应该高亮显示（绿色背景）
   - 复选框应该被选中
   - 再次点击，应该取消选中

8. **测试"全选"按钮**
   - 点击"全选"按钮
   - 所有行应该被选中并高亮

9. **测试"清除"按钮**
   - 点击"清除"按钮
   - 所有行应该取消选中

10. **选择有效的训练动作行**
    - 选择以下行：
      - "胸部训练"
      - "坐姿下斜推 3组 12次 组内暂停 10+5 休息 60"
      - "上斜推 3组 12次 休息 60"
      - "平板推 3组 12次 休息 60"
      - "飞鸟 3组 15次 休息 45"
      - "绳索夹胸 3组 15次 休息 45"
      - "俯卧撑 3组 力竭 休息 60"
    - 不要选择无关的文字行

11. **点击"确认创建"按钮**
    - 弹窗应该关闭
    - 应该看到新的弹窗显示解析结果
    - 验证解析的计划是否正确

12. **保存计划**
    - 点击"保存"按钮
    - 应该看到"计划已保存"的提示
    - 计划应该出现在列表中

### 测试 2：从"查看结果"进入手动选择

1. **重复测试 1 的步骤 1-4**

2. **点击"查看结果"按钮**
   - 应该看到识别结果弹窗
   - 显示解析的计划摘要

3. **点击"选项"按钮**
   - 应该看到操作菜单
   - 第一个选项应该是"手动选择文字"

4. **点击"手动选择文字"**
   - 应该进入文字选择界面
   - 继续测试 1 的步骤 6-12

### 测试 3：测试取消操作

1. **进入文字选择界面**（参考测试 1）

2. **点击"取消"按钮**
   - 弹窗应该关闭
   - 返回到自定义计划列表

3. **点击弹窗外的遮罩层**
   - 弹窗应该关闭
   - 返回到自定义计划列表

### 测试 4：测试空选择

1. **进入文字选择界面**（参考测试 1）

2. **不选择任何文字行**

3. **点击"确认创建"按钮**
   - 应该看到提示："请至少选择一行文字"
   - 弹窗不应该关闭

4. **选择至少一行文字后再点击"确认创建"**
   - 应该正常进入解析流程

### 测试 5：测试标准格式（不触发手动选择）

1. **点击"📷 扫描图片"**

2. **选择"全身力量训练（4个动作）"**
   - 这是标准格式，质量评分应该很高
   - 应该直接显示识别结果，不提示手动选择

3. **点击"选项"按钮**
   - 应该看到"手动选择文字"选项
   - 点击后应该能进入手动选择界面

## 预期结果

### 功能正常的标志：
✅ 低质量文字自动触发手动选择提示
✅ 文字行正确显示，每行有复选框
✅ 点击行可以切换选中状态
✅ 选中的行有绿色高亮
✅ "全选"和"清除"按钮正常工作
✅ 确认创建后正确解析选中的文字
✅ 取消操作正常工作
✅ 空选择有提示
✅ 从"选项"菜单也能进入手动选择

### 常见问题排查：

#### 问题 1：点击"手动选择"后没有显示弹窗
**检查点：**
- 打开控制台，查看是否有 `showTextSelection 开始` 的日志
- 检查 `ocrTextLines` 数组是否有数据
- 检查 `showTextSelectionModal` 是否为 `true`

**解决方法：**
- 查看控制台日志，确认文字是否正确分割
- 检查 WXML 中的 `wx:if="{{showTextSelectionModal}}"` 是否正确

#### 问题 2：文字行显示不完整
**检查点：**
- 查看控制台中的"分割后的行"日志
- 检查原始文字是否包含换行符

**解决方法：**
- 确认文字按 `\n` 或 `\r` 正确分割
- 检查 `filter(line => line.trim())` 是否过滤了空行

#### 问题 3：选中状态不更新
**检查点：**
- 点击行时是否触发 `toggleTextLineSelection` 函数
- 检查 `data-index` 是否正确传递

**解决方法：**
- 在 `toggleTextLineSelection` 中添加 `console.log` 查看是否被调用
- 检查 WXML 中的 `data-index="{{index}}"` 是否正确

#### 问题 4：确认创建后解析失败
**检查点：**
- 查看选中的文字内容
- 检查 `ocrParser.parseTrainingPlan` 是否正常工作

**解决方法：**
- 在 `confirmTextSelection` 中添加日志查看选中的文字
- 测试解析器是否能正确解析选中的文字

## 调试技巧

### 1. 查看控制台日志
打开微信开发者工具的控制台，查看以下关键日志：
- `=== showRecognizedPlan 开始 ===`
- `是否应该显示手动选择: true/false`
- `=== showTextSelection 开始 ===`
- `分割后的行数: X`
- `文字选择弹窗已显示`

### 2. 检查数据状态
在控制台中输入以下命令查看数据：
```javascript
// 查看当前页面数据
getCurrentPages()[getCurrentPages().length - 1].data
```

### 3. 手动触发函数
在控制台中手动调用函数测试：
```javascript
const page = getCurrentPages()[getCurrentPages().length - 1];
page.showTextSelection('测试文字\n第二行\n第三行');
```
