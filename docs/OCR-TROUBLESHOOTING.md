# OCR功能故障排除指南

## 🐛 常见错误及解决方案

### 错误1：evaluateOCRQuality is not a function

**错误信息**：
```
TypeError: ocrParser.evaluateOCRQuality is not a function
```

**原因**：
- 微信开发者工具缓存问题
- 模块未正确加载
- 文件未保存

**解决方案**：

#### 方案1：清除缓存（推荐）
1. 点击开发者工具菜单：项目 → 清除缓存
2. 选择"清除文件缓存"和"清除授权数据"
3. 重新编译项目
4. 刷新页面

#### 方案2：重启开发者工具
1. 完全关闭微信开发者工具
2. 重新打开项目
3. 等待编译完成
4. 测试功能

#### 方案3：手动刷新
1. 在开发者工具中按 `Ctrl + R`（Windows）或 `Cmd + R`（Mac）
2. 强制刷新页面
3. 测试功能

#### 方案4：检查文件
1. 确认 `utils/ocr-parser.js` 文件已保存
2. 确认文件末尾有正确的 `module.exports`：
```javascript
module.exports = {
  parseTrainingPlan,
  formatPlanSummary,
  evaluateOCRQuality
};
```
3. 确认 `pages/custom-plans/custom-plans.js` 正确引入：
```javascript
const ocrParser = require('../../utils/ocr-parser.js')
```

#### 方案5：使用真机调试
1. 如果开发者工具仍有问题，使用真机调试
2. 扫码预览
3. 在真机上测试功能

**预防措施**：
- 代码已添加fallback机制，即使函数不可用也不会报错
- 如果遇到此错误，功能会使用默认评分（80分）

---

### 错误2：wx.ocr.recognize is not a function

**错误信息**：
```
TypeError: Cannot read property 'recognize' of undefined
```

**原因**：
- 微信开发者工具不支持OCR API
- 需要在真机上测试

**解决方案**：

#### 自动fallback（已实现）
代码已自动处理此问题：
```javascript
// 检查OCR API是否可用
if (!wx.ocr || typeof wx.ocr.recognize !== 'function') {
  console.log('OCR API不可用，使用模拟识别');
  this.simulateOCR(imagePath);
  return;
}
```

#### 使用模拟识别
1. 点击"扫描图片"
2. 选择预设的训练计划：
   - 全身力量训练（4个动作）
   - HIIT燃脂训练（4个动作）
   - 臀腿训练（5个动作）
   - 上肢训练（6个动作）
3. 或选择"手动输入文字"

#### 真机测试
1. 使用真机预览
2. 扫码打开小程序
3. 测试OCR识别功能
4. 真机支持完整的OCR API

### 错误3：识别质量评分为0分

**症状**：
- 识别完成但评分为0
- 显示多个问题

**原因**：
- 图片质量太差
- OCR识别失败
- 文本格式不标准

**解决方案**：

#### 检查原始文字
1. 点击"选项"
2. 选择"查看原始文字"
3. 查看OCR识别的内容
4. 判断是OCR问题还是解析问题

#### 改进图片质量
1. 确保文字清晰可见
2. 使用打印文字（不要手写）
3. 光线充足，避免阴影
4. 背景简洁，对比度高
5. 每行一个动作

#### 使用标准格式
```
动作名 3组 x 12次 休息60秒
```

#### 查看识别建议
1. 点击"选项"
2. 选择"查看识别建议"
3. 根据建议改进图片
4. 重新识别

### 错误4：动作名称显示为"动作1"、"动作2"

**症状**：
- 识别完成但动作名称不正确
- 显示为通用名称

**原因**：
- 动作名称不在识别库中
- OCR识别错误
- 格式不标准

**解决方案**：

#### 检查支持的动作
查看支持的动作列表：
- 下肢：深蹲、硬拉、弓步蹲、臀桥、腿举
- 上肢：卧推、引体向上、俯卧撑、肩推、侧平举、二头弯举、三头屈伸
- 核心：平板支撑、仰卧起坐、卷腹、俄罗斯转体
- 有氧：开合跳、波比跳、高抬腿、登山跑、跳绳、跑步
- 瑜伽：猫牛式、下犬式、战士式、树式、婴儿式

#### 使用标准名称
- 使用列表中的标准名称
- 或使用常见别名（蹲起→深蹲）

#### 手动修改
1. 点击"选项"
2. 选择"修改计划"
3. 手动修改动作名称
4. 保存计划

#### 查看原始文字
1. 点击"选项"
2. 选择"查看原始文字"
3. 确认OCR识别的动作名称
4. 如果OCR正确但未识别，可能需要添加到动作库

### 错误5：组数次数识别错误

**症状**：
- 组数或次数不正确
- 使用了默认值

**原因**：
- 数字格式不标准
- OCR识别错误
- 多个数字混淆

**解决方案**：

#### 使用标准格式
推荐格式：
```
深蹲 3组 x 12次 休息60秒
深蹲 3x12 60s
深蹲 3*12 60
```

#### 确保数字清晰
- 字体大小：14-18号
- 颜色：黑色
- 背景：白色
- 对比度高

#### 查看提取日志
打开控制台查看：
```
提取到的数字: 3, 12, 60
识别组数: 3
识别次数/时长: 12
识别休息时间: 60秒
```

#### 手动修改
1. 点击"选项"
2. 选择"修改计划"
3. 手动修改组数、次数
4. 保存计划

### 错误6：休息时间使用默认值

**症状**：
- 休息时间不是图片中的值
- 使用了智能默认值

**原因**：
- 没有明确标注休息时间
- 格式不标准
- OCR识别错误

**解决方案**：

#### 明确标注休息时间
推荐格式：
```
深蹲 3组 x 12次 休息60秒
深蹲 3x12 60s
深蹲 3*12 rest 60
```

#### 理解智能默认值
如果没有识别到休息时间，系统会根据动作类型设置：
- 深蹲、硬拉、卧推、引体向上：90秒
- 开合跳、波比跳、高抬腿、登山跑：30秒
- 平板支撑、仰卧起坐、卷腹：45秒
- 其他动作：60秒

#### 手动修改
1. 点击"选项"
2. 选择"修改计划"
3. 手动修改休息时间
4. 保存计划

## 🔧 调试技巧

### 技巧1：查看控制台日志

打开控制台查看详细日志：
```
=== 开始解析训练计划 ===
原始文字: ...
共5行待处理
处理第1行: "深蹲 3组 x 12次 休息60秒"
提取到的数字: 3, 12, 60
精确匹配到: 深蹲 -> 深蹲
识别组数: 3
识别次数/时长: 12
识别休息时间: 60秒
识别动作: 深蹲 - 3组 x 12次, 休息60秒
解析结果: ...
=== 解析完成 ===
```

### 技巧2：使用调试功能

识别后的调试选项：
1. **查看原始文字**：查看OCR识别的内容
2. **查看识别建议**：查看质量评分和改进建议
3. **重新解析**：使用相同文字重新解析
4. **修改计划**：手动编辑识别结果

### 技巧3：分步测试

1. **测试OCR**：先查看原始文字，确认OCR是否正确
2. **测试解析**：如果OCR正确，点击"重新解析"测试解析逻辑
3. **测试保存**：确认解析正确后，测试保存功能
4. **测试训练**：保存后，测试训练功能

### 技巧4：对比测试

1. 使用模拟数据测试（应该100%成功）
2. 使用手动输入测试（应该95%+成功）
3. 使用图片识别测试（应该85%+成功）
4. 对比结果，找出问题环节

## ✅ 预防措施

### 开发阶段

1. **定期清除缓存**
   - 每次修改代码后清除缓存
   - 避免缓存导致的问题

2. **使用真机测试**

   - 开发者工具仅供参考

   - 提供友好的错误提示

4. **添加fallback机制**
