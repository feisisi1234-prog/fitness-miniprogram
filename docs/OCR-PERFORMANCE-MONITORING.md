# OCR 扫描图片性能监控文档

## 概述

本文档详细说明了自定义计划页面中"扫描图片"功能的完整流程、各个步骤的耗时监控，以及性能优化建议。

## 完整流程图

```
用户点击"📷 扫描图片"
    ↓
[步骤0] 显示拍照提示弹窗
    ↓
用户点击"继续"
    ↓
[步骤1] 选择/拍摄图片 (wx.chooseImage)
    ↓
[步骤2] OCR 文字识别 (wx.ocr.recognize 或模拟)
    ↓
[步骤3] 提取识别的文字 (extractTextFromOCR)
    ↓
[步骤4] 解析训练计划 (ocrParser.parseTrainingPlan)
    ↓
[步骤5] 质量评估 (ocrParser.evaluateOCRQuality)
    ↓
[决策点] 质量是否良好？
    ├─ 是 → 显示识别结果
    └─ 否 → 提示手动选择文字
         ↓
    用户选择"手动选择"或"查看结果"
         ↓
    [可选] 手动选择文字行
         ↓
    重新解析选中的文字
         ↓
显示最终结果
    ↓
用户点击"保存"
    ↓
[步骤6] 保存计划到本地存储 (wx.setStorageSync)
    ↓
完成 ✅
```

## 性能监控点

### 1. 流程开始
**位置**: `scanImagePlan()` 函数开始
**日志**: 
```
🚀 [性能监控] 扫描图片流程开始
```
**记录**: `_flowStartTime` - 流程开始时间戳

### 2. 步骤1: 选择图片
**API**: `wx.chooseImage()`
**功能**: 从相册选择或拍摄照片
**日志**:
- 开始: `⏱️  [性能监控] 步骤1: 开始选择图片`
- 成功: `✅ [性能监控] 步骤1完成: 选择图片耗时 XXXms`
- 失败: `❌ [性能监控] 步骤1失败: 选择图片耗时 XXXms`

**预期耗时**: 
- 从相册选择: 1000-3000ms (取决于用户操作速度)
- 拍摄照片: 2000-5000ms (取决于用户操作速度)

### 3. 步骤2: OCR 识别
**API**: `wx.ocr.recognize()` (真机) 或 模拟识别 (开发工具)
**功能**: 识别图片中的文字
**日志**:
- 开始: `⏱️  [性能监控] 步骤2: 开始OCR识别`
- 成功: `✅ [性能监控] 步骤2完成: OCR识别耗时 XXXms`
- 失败: `❌ [性能监控] 步骤2失败: OCR识别耗时 XXXms`
- API不可用: `⚠️  [性能监控] OCR API不可用检测耗时 XXXms，使用模拟识别`

**预期耗时**:
- 真机 OCR: 500-2000ms (取决于图片大小和复杂度)
- 模拟识别: 0-100ms (用户选择模拟选项的时间)

### 4. 步骤3: 提取文字
**函数**: `extractTextFromOCR()`
**功能**: 从 OCR 结果中提取和排序文字
**日志**:
- 开始: `⏱️  [性能监控] 步骤3: 开始提取文字`
- 完成: `✅ [性能监控] 步骤3完成: 提取文字耗时 XXXms`

**预期耗时**: 10-50ms (取决于识别的文字量)

### 5. 步骤4: 解析训练计划
**函数**: `ocrParser.parseTrainingPlan()`
**功能**: 解析文字，提取动作、组数、次数、休息时间
**日志**:
- 开始: `⏱️  [性能监控] 步骤4: 开始解析训练计划`
- 完成: `✅ [性能监控] 步骤4完成: 解析训练计划耗时 XXXms`

**预期耗时**: 20-100ms (取决于文字量和复杂度)

### 6. 步骤5: 质量评估
**函数**: `ocrParser.evaluateOCRQuality()`
**功能**: 评估识别质量，决定是否需要手动选择
**日志**:
- 开始: `⏱️  [性能监控] 步骤5: 开始质量评估和显示结果`
- 完成: `✅ [性能监控] 步骤5完成: 质量评估耗时 XXXms`
- 总耗时: `🏁 [性能监控] 完整流程总耗时: XXXms (X.XX秒)`

**预期耗时**: 5-20ms

### 7. 步骤6: 保存计划
**API**: `wx.setStorageSync()`
**功能**: 保存计划到本地存储
**日志**:
- 开始: `⏱️  [性能监控] 步骤6: 开始保存计划`
- 完成: `✅ [性能监控] 步骤6完成: 保存计划耗时 XXXms`
- 最终总耗时: `🎉 [性能监控] 从扫描到保存完整流程总耗时: XXXms (X.XX秒)`

**预期耗时**: 10-50ms

## 性能瓶颈分析

### 主要耗时点
1. **用户操作时间** (不可控)
   - 选择/拍摄图片: 1-5秒
   - 手动选择文字: 3-10秒

2. **OCR 识别** (可优化)
   - 真机识别: 0.5-2秒
   - 优化方向: 图片压缩、预处理

3. **文字解析** (已优化)
   - 当前: 20-100ms
   - 性能良好

## 优化建议

### 1. 图片预处理
- 在调用 OCR 前对图片进行压缩和优化
- 调整图片对比度和亮度
- 预期提升: 减少 OCR 识别时间 20-30%

### 2. 缓存机制
- 缓存已识别的图片结果
- 避免重复识别相同图片
- 预期提升: 重复识别时间接近 0ms

### 3. 异步优化
- 使用 Promise 封装异步操作
- 并行处理可以并行的任务
- 预期提升: 减少总耗时 10-15%

### 4. 解析器优化
- 优化正则表达式匹配
- 减少不必要的循环
- 预期提升: 减少解析时间 30-40%

## 如何查看性能日志

1. 打开微信开发者工具
2. 点击"控制台"标签
3. 点击"📷 扫描图片"按钮
4. 在控制台中查看带有 `[性能监控]` 标记的日志
5. 关注以下关键指标：
   - 各步骤耗时
   - 完整流程总耗时
   - 是否触发手动选择
