# 食物识别功能快速开始

## 🚀 5分钟快速配置

### 步骤 1: 获取 Coze API Key

1. 访问 https://www.coze.cn/
2. 注册/登录账号
3. 进入"个人中心" -> "API Keys"
4. 创建新的 API Key
5. 复制 API Key（格式：`pat_xxxxx...`）

### 步骤 2: 创建 Bot

1. 在 Coze 平台点击"创建 Bot"
2. 选择"空白 Bot"
3. 配置 Bot 信息：
   - 名称：食物识别助手
   - 描述：识别食物并提供营养信息
4. 在"能力配置"中启用"视觉模型"
5. 在"提示词"中输入：

```
你是一个专业的营养师助手。当用户上传食物图片时，请识别食物并提供以下信息：

1. 食物名称
2. 热量（千卡/100g）
3. 升糖指数（GI值，0-100）
4. 升糖负荷（GL值）
5. 食用建议

请严格按照以下格式返回：

食物名称: [食物名称]
热量: [数值]千卡/100g
升糖指数(GI): [数值]
升糖负荷(GL): [数值]
食用建议: [详细的食用建议，包括适合人群、食用时间、注意事项等]

示例：
食物名称: 苹果
热量: 52千卡/100g
升糖指数(GI): 36
升糖负荷(GL): 6
食用建议: 苹果是低GI食物，富含膳食纤维和维生素C。适合减肥和控糖人群食用。建议在两餐之间作为加餐，每次食用1个中等大小的苹果（约150g）。糖尿病患者可以适量食用，但需注意监测血糖。
```

6. 发布 Bot
7. 复制 Bot ID（在 Bot 详情页面）

## 🐛 常见问题

### 问题 1: 文件上传失败
```
错误信息: 文件上传失败: HTTP 401
```
**原因**: API Key 配置错误
**解决**: 检查 API Key 是否正确复制，确保包含 `pat_` 前缀

### 问题 2: Chat 创建失败
```
错误信息: Chat请求失败: bot not found
```
**原因**: Bot ID 配置错误或 Bot 未发布
**解决**: 
1. 检查 Bot ID 是否正确
2. 确认 Bot 已发布
3. 确认 Bot 启用了视觉模型

### 问题 3: 识别超时
```
错误信息: 识别超时，请重试
```
**原因**: AI 处理时间过长
**解决**: 
1. 检查网络连接
2. 尝试使用更小的图片
3. 增加轮询次数（修改代码中的 `retryCount > 30`）

### 问题 4: 结果解析失败
```
显示: 解析失败
```
**原因**: Bot 返回格式不符合预期
**解决**: 
1. 检查 Bot 的提示词是否正确
2. 查看控制台的原始返回内容
3. 调整解析逻辑
